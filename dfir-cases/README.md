# DFIR Case Studies

This directory contains simulated Digital Forensics and Incident Response (DFIR) investigations designed for educational and research purposes.

The cases included here aim to demonstrate investigative methodology, evidence analysis, and incident response workflows used by security analysts and forensic investigators.

All cases follow a simplified version of the **IPED forensic methodology**:

Identification → Preservation → Examination → Documentation

## Investigation Methodology

### 1. Identification
The first step involves identifying suspicious activity or potential indicators of compromise (IOCs).  
Examples may include:

- suspicious authentication logs
- unusual network traffic
- malware indicators
- abnormal system behavior

### 2. Preservation
Relevant artifacts are collected and preserved to prevent evidence tampering.  
Examples include:

- log files
- memory artifacts
- network captures (PCAP)
- file hashes

All artifacts in this repository are **synthetic or sanitized**.

### 3. Examination
Collected evidence is analyzed using forensic techniques and tooling.  
Typical analysis tasks include:

- timeline reconstruction
- IOC correlation
- log analysis
- hash verification
- artifact analysis

### 4. Documentation
Each investigation includes structured documentation describing:

- incident overview
- timeline of events
- identified indicators of compromise
- forensic findings
- recommendations

## Purpose

These DFIR case simulations are intended to demonstrate:

- investigative thinking
- incident response methodology
- security analysis workflows
- documentation practices used in professional environments

All investigation data contained in this directory is **synthetic or anonymized** and is included solely for educational purposes.
